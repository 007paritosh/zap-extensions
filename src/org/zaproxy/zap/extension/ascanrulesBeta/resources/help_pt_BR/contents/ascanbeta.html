Regras de varredura ativa - beta

Este complemento contém as seguintes regras de varredura ativa em qualidade beta:

Revelação de arquivos de Backup

Busca cópias de arquivos de backup no web server com nomes de arquivo triviais, que podem revelar informações confidenciais.

Erro de configuração de domínios cruzados

Verifica se o servidor web está configurado para permitir acesso Cross Domain, um serviço de terceiros mal-intencionado, por exemplo. Atualmente checa se há caracteres curinga no arquivo crossdomain.xml da Adobe e em clientaccesspolicy.xml do SilverLight.

Token de CSRF

Verifica a existência de tokens de anti-CSRF. Alerta em solicitações que não parecem conter tokens anti-CSRF.

Injeção de linguagem de expressão

Verifica se o aplicativo da web está sujeita a ataques de injeção de linguagem de expressão (EL), que ocorrem quando um aplicativo não consegue neutralizar suficientemente elementos especiais que poderiam modificar a instrução EL pretendida antes de ser executado.

Vulnerabilidade do Heartbleed OpenSSL

Detecta se o servidor web é vulnerável a vulnerabilidade do OpenSSL Heartbleed, explorando isso. Para mais detalhes consulte CVE-2014-0160.

Poluição do parâmetro HTTP (HPP)

Fornece duplicados (ou numerosos) parâmetros HTTP com o mesmo nom, o que pode causar um aplicativo ou site interpretar valores de maneiras inesperadas. Aproveitando estes efeitos, um indivíduo mal-intencionado pode ser capaz de ignorar a validação de entrada, provocar erros ou modificar os valores de variáveis internos. Há diferença no tratamento dos parâmetros duplicados, impactando os clientes (navegadores) e servidores.

Método de HTTP inseguro

Detecta (e explora, dependendo das configurações de varredura) conhecidos métodos de HTTP inseguros, habilitados para a URL.

Erro de estouro de número inteiro

Procura por indicadores de estouros de número inteiro no código compilado, o que faz com que o servidor web cair. Ele faz isso, colocando para fora várias cadeias de caracteres de inteiros projetados para tentar estimular respostas ruins.

Padding Oracle

Esta varredura tenta manipular o padding de strings criptografadas para desencadear uma resposta de erro indicando uma provável vulnerabilidade de padding Oracle. Tal uma vulnerabilidade pode afetar qualquer aplicativo ou framework que usa criptografia incorretamente, como algumas versões do ASP.net, Java Server Faces e Mono.

Execução remota de código - CVE-2012-1823

Detecta CVE-2012-1823 para fazer execução remota de código em servidor web com base em PHP-CGI.

Fixação de sessão

Fixação de sessão pode ser possível. Se esse problema ocorre com uma URL de login (onde o usuário se autentica para o aplicativo), então a URL pode ter sido dada a uma vítima por um invasor, juntamente com uma identificação de sessão fixa, para depois assumir a identidade da vítima usando a identificação de determinada sessão. Se o problema ocorre com uma página que não é de login, a URL e o id de sessão fixa podem ser usados por um invasor somente para controlar as ações de um usuário não-autenticado. Se a vulnerabilidade ocorre em um campo de cookie ou num de formulário (parâmetro POST) em vez de ocorrer num parâmetro de URL (GET), então alguma outra vulnerabilidade pode também ser necessária para utilização do campo de cookie no navegador da vítima, para então permitir que a vulnerabilidade seja explorada.

ShellShock - CVE-2014-6271

Esta varredura executa dois ataques para detectar servidores vulneráveis a CVE-2014-6271, também conhecido como ShellShock.
O primeiro é um simples ataque refletido e o segundo é um ataque de tempo determinado.

Exposição de código de fonte - CVE-2012-1823

Exploit CVE-2012-1823 para expor o código-fonte no lado do servidor PHP, em um servidor de web com base em PHP-CGI.

Exposição de código fonte - SVN

Usa os metadados de repositório do código fonte Subversion para procurar por arquivos contendo código-fonte no servidor web.

Exposição de código fonte - /WEB-INF

Explora a presença de uma pasta /WEB-INF desprotegida para baixar e descompilar classes Java, para exibir o código-fonte Java.

SQL Injection - Hypersonic, baseada em tempo

Esta varredura usa sintaxe específica de SQL Hypersonic para tentar induzir atrasos na declaração SQL chamada pela página.
Se a query não-modificada não é afetada pelo retardo, e se o retardo na query modificada pode ser controlado, isso é indicativo de uma vulnerabilidade de injeção SQL baseada em um banco de dados Hypersonic SQL.
Esta varredura é sensível a tempo e só deve ser usada em uma tentativa de encontrar vulnerabilidades de injeção SQL difíceis e não óbvias em um banco de dados Hypersonic suspeito.
Por esta razão, o número de threads de varredura ativa deve ser definido para o mínimo, para minimizar a carga sobre o servidor web, servidor de aplicativos e banco de dados, a fim de evitar falsos positivos causados por atrasos de carga em vez de atrasos de injeção de SQL.
A varredura testa somente vulnerabilidades de injeção SQL baseadas em tempo.

Enumeração de usernames

Pode ser possível fazer enumeração de nomes de usuário com base em diferentes respostas HTTP quando usernames válidos e inválidos são fornecidos. Isso aumenta significativamente a probabilidade de sucesso com ataques de brute force de senha contra o sistema. Observe que falsos positivos às vezes podem ser minimizados aumentando a opção 'Força de ataque' no ZAP. Por favor verificar manualmente o campo 'Other Info' para confirmar se este é realmente um problema. Este scanner é ignorada se há que nenhum contextos definidos que usam autenticação baseada em formulário e só funciona contra o URL identificado como a URL de login de um contexto.

Injeção de XPath

Conforme descrito pelo OWASP: "ataques de injeção de XPath ocorrem quando um site da web usa informações fornecidas pelo usuário para construir uma consulta XPath para dados XML. Enviando informações intencionalmente malformadas em site da web, um invasor pode descobrir como os dados XML são estruturados, ou dados de acesso que ele normalmente não pode ter acesso. Ele pode até mesmo ser capaz de elevar seus privilégios no site se os dados XML está sendo usados para autenticação (como um arquivo de usuário baseada em XML) ou autorização." Este scanner tenta identificar tais fraquezas.

XXE

Este componente tenta identificar aplicações sujeitas a ataques externos de entidade (XXE) XML. Aplicativos que fazem parsing de XML de entrada podem estar sujeitas a XXE quando parsers fracamente ou mal configurados analisadores manipulam a entrada em XML contndo a referência a uma entidade externa, tal como um arquivo local, solicitações HTTP para sistemas internos ou terciários, etc.
